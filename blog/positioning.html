Just what is the differences between relative, absolute, and fixed positioning?

TECHNICAL


written by Justin Cole

Nov 14 2020

Admittedly, CSS is definitely one of my weak points. That is why through these technical blogs I intend on making my mastery of CSS the focus. To try and help myself to understand all the different properties and attributes and hopefully illuminate what currently seems to be a frustrating roadblock.


Firstly, I should start by mentioning static positioning. This is the default position attribute each element will have if nothing has been specified. With static positioning each element follows directly after one another in their natural order (case in point, these paragraphs I am typing).



Relative Position


Relative position allows us to make use of the Top, Bottom, Right and Left properties. With our element set to relative position we can now move the element in relation to it's original position, say Top: 10px; and Left: 20px;. This will move the element 10 pixels down from it's top and 20 pixels right of it's left side (confusing, I know).



Absolute Position


Position absolute removes the element from the normal document flow and allows for stacking of elements used with the z-index property. The element is now able to be positioned in relation to it's closet positioned ancestor. If no ancestor has a position property other than default, the element will be positioned in relation to the document body. Similar to position relative, the Top, Bottom, Left and Right properties inform the magnitude.




Fixed Position


Like position absolute, position fixed removes the element from the normal document flow. It is now always fixed in relation to the viewport. Take the soon to be navigation bar at the top of this page. Scrolling does nothing to affect the position of this element. It is always locked in position at the top of the viewport. Once again the Top, Bottom, Left and Right properties can be used to inform where this element will be fixed.
